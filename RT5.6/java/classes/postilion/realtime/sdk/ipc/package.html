This packages encapsulates a generic communications infrastructure.

Communications in the Postilion Realtime Framework is built on top of essentially two abstractions, namely saps (Service Access Points) and endpoints.

Saps are essentially Endpoint factories. Two types of sap are defined: a client and a server sap. They differ in the state of the endpoint they produce. A client sap produces disconnected endpoints that must then be connected to a remote entity. A server sap produces endpoints that are already connected and represent a newly established connection.
